Реализация МГУА на языке C

Метод группового учета аргументов для моделирования качества воды

Проект: GMDH (Group Method of Data Handling)
Дата: Ноябрь 2025
Язык: C99, ~1600 строк
Данные: 595 образцов × 38 признаков

Суть проекта

Реализованы классические советские алгоритмы индуктивного моделирования (А. Г. Ивахненко, 1960-е) на современном C для автоматического построения математических моделей. Применение: прогнозирование pH воды на очистных сооружениях.

Реализованные алгоритмы

Алгоритм Принцип работы Сложность
Комбинаторный МГУА Полный перебор всех пар признаков, построение квадратичных полиномов, выбор лучших по внешнему критерию O(n²)
Многорядный МГУА Послойная эволюция: каждый слой создает новые признаки из лучших моделей предыдущего ("скрещивание родителей") O(n·k·l)
Базовая модель: y = a₀ + a₁x₁ + a₂x₂ + a₃x₁² + a₄x₂² + a₅x₁·x₂

Метод оценки: Метод наименьших квадратов (МНК) с решением системы нормальных уравнений методом Гаусса.

Критерий регулярности: Среднеквадратичная ошибка (RMSE) на валидационной выборке (30% данных).

Результаты на реальных данных

Задача: Предсказание pH на выходе очистных сооружений (pH_tank3)

Набор данных: Водоочистная станция, январь–март 1990, 595 наблюдений

Лучшая модель (многорядный МГУА, слой 1):

✓ RMSE = 0.30 единиц pH (R² = 0.993)

✓ Точность: ±0.3 pH от истинного значения (диапазон 7.3–7.6)

pH = 357.12 + 34.46·f₁ - 126.32·f₂ + 0.087·f₁² + 10.46·f₂² - 4.51·f₁·f₂

где f₁, f₂ — производные признаки из выходов слоя 0

0.30 RMSE (pH units)
70/30 Train/Valid split
7/7 Тесты пройдены

Технические детали

  • Модули: data.c (загрузка CSV, обработка пропусков), polynomial.c (МНК регрессия), gmdh_combinatorial.c, gmdh_multirow.c
  • Тестирование: 7 модульных тестов покрывают все компоненты
  • Сборка: Makefile с раздельными директориями build/ и bin/
  • Компиляция: gcc -Wall -Wextra -O2 -std=c99
  • Зависимости: только стандартная библиотека C + libm

Интерпретация результатов

Физический смысл: Модель автоматически обнаружила, что выходной pH зависит от комбинации входных параметров воды (расход, жесткость, температура, проводимость). Ошибка ±0.3 pH соответствует точности лабораторных pH-метров.

Преимущества МГУА:

  • Автоматический отбор признаков и построение модели
  • Защита от переобучения через внешний критерий (валидация)
  • Интерпретируемые полиномиальные модели
  • Не требует предположений о виде зависимости

Применение

Реализованные алгоритмы применимы для:

  • Моделирования технологических процессов
  • Прогнозирования временных рядов
  • Автоматического feature engineering в ML
  • Задач, где нужна интерпретируемость модели
